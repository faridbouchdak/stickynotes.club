# Product Playbook

> Status: v1

## Purpose

Translate the Constitution into daily product decisions.

## Product workflow

1.  Start with the user problem.
2.  Describe the desired behaviour.
3.  Write the Help Center page.
4.  Build the feature.
5.  Test.
6.  Improve.

## Documentation rules

-   One question per page.
-   Start with the user's goal.
-   Short paragraphs.
-   Related guides.
-   Stable permalinks.

## UX rules

-   Prefer recognition over explanation.
-   Prefer calm over clever.
-   Add complexity only when proven valuable.

## Writing

-   Friendly.
-   Practical.
-   Human.
-   No hype.
-   Explain outcomes before actions.

## Feature checklist

-   Solves a real problem.
-   Fits Public or Private.
-   Adds minimal complexity.
-   Respects KISS.
-   Can be documented clearly.

## Reflection

After every release: - update docs; - capture discoveries; - promote
recurring patterns to this Playbook.
